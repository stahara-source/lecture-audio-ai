# 講義音声ノートAI (Lecture Audio AI)

## 1. プロジェクト概要
大学の講義や自習の録音ファイルをアップロードし、Gemini APIと連携して「文字起こし」「3行要約」「要点抽出」を自動化するスマホ対応Webアプリです。ノート作成の時間を削減し、復習効率を最大化することを目的としています。

## 2. 解決する課題とアプローチ
* **課題：講義中のメモが追いつかず、内容に集中できない**
    * **解決策**: Gemini API（2.0 Flash）を活用し、高精度な文字起こしを自動生成します。
* **課題：録音を見返す時間がなく、復習が後回しになる**
    * **解決策**: 全文だけでなく「3行要約」と「箇条書きの要点」を同時に出力し、一目で内容を把握可能にします。
* **課題：文字起こししたデータの転記が面倒**
    * **解決策**: 各結果に「コピーボタン」を実装し、他のノートアプリへの貼り付けを容易にします。

## 3. 主な機能
* **APIキー設定**: ユーザー独自のGemini APIキーをブラウザ（localStorage）に保存。
* **音声解析**: 音声ファイルをGemini 2.0 Flashで解析。
* **3要素の出力**: 文字起こし全文、3行要約、重要ポイントをカード形式で表示。
* **クリップボード連携**: 各解析結果をワンクリックでコピー可能。

## 4. 業務フロー（ユーザー導線）
1. **APIキー入力**: 右上のギアアイコンからAPIキーを登録。
2. **題名を付ける**: 講義名などを入力。
3. **音声ファイル選択**: 解析したい音声ファイルを選択。
4. **Gemini要約する**: 「解析開始」ボタンをクリック。
5. **結果の確認**: 画面に表示された要約等を確認・コピー。

## 5. セットアップ手順
1. **GitHub Pagesの有効化**
    * リポジトリの Settings > Pages から、Branchを「main」に設定して保存（Save）します。
2. **APIキーの設定**
    * 公開されたURLにアクセスし、設定画面からGemini APIキーを入力・保存します。

## 6. 使用技術スタック
* **フロントエンド**: HTML5, CSS3, JavaScript
* **AI API**: Google Gemini API (models/gemini-2.0-flash)
* **ホスティング**: GitHub Pages

* **音声解析**: 正常（30秒の音声で文字起こし・要約・要点の出力を確認）
* **コピー機能**: 正常（各項目がクリップボードに保存されることを確認）
